# Release Artifact Verification

## Status

Reviewer instructions for verifying public-safe release artifacts.

These instructions do not claim production certification.

## Release Artifacts

Expected release artifacts:

- `maci-elyria-review-bundle.zip`
- `maci-elyria-review-bundle.zip.sha256`
- `sbom.json`
- `release-provenance.json`
- SLSA/in-toto provenance generated by the SLSA workflow

## Verify the Review Bundle Hash

From the directory containing the downloaded artifacts:

```bash
sha256sum -c maci-elyria-review-bundle.zip.sha256
```

Expected result:

```text
maci-elyria-review-bundle.zip: OK
```

## Inspect the SBOM

```bash
jq . sbom.json
```

The SBOM is public-safe and lists repository source files and declared Python requirements for review.

## Inspect Candidate Release Provenance

```bash
jq . release-provenance.json
```

The candidate provenance records:

- commit
- review bundle digest
- bundle SHA file digest
- SBOM digest
- signature record
- claim boundary

## Verify SLSA Provenance

Install or download the SLSA verifier from the official SLSA verifier release path.

Then verify the downloaded artifact against this source repository:

```bash
slsa-verifier verify-artifact maci-elyria-review-bundle.zip \
  --provenance-path multiple.intoto.jsonl \
  --source-uri github.com/Kamanaka5502/maci-elyria-seam-pilot
```

If the provenance file name differs, replace `multiple.intoto.jsonl` with the generated SLSA provenance file.

## Verify No Certification Overclaim

A valid release verification result proves artifact origin and integrity.

It does not prove:

```text
production certified
regulatory certified
customer security approved
external audit approved
commercial deployment rights without written scope
public exposure of private Elyria/VERITA kernel
```

## Correct Claim

```text
Release artifacts have verification instructions, SBOM, candidate provenance, and SLSA provenance generation path for reviewer trust evaluation.
```
